# 07 — 部署运维指南

## 推荐部署平台

| 平台 | 免费额度 | 适合场景 |
|------|----------|----------|
| **Railway** | $5/月额度 | 支持 SQLite 持久化，推荐 |
| Vercel | 慷慨免费 | 适合纯前端，SQLite 需额外配置 |

> **首选 Railway**：它提供持久化磁盘，SQLite 数据库文件不会丢失。Vercel 的 Serverless 环境下 SQLite 文件可能被回收。

## Railway 部署步骤

### 1. 准备工作

确保项目根目录有以下文件：
- `package.json`（含 `build` 和 `start` 脚本）
- `prisma/schema.prisma`

`package.json` 需包含：
```json
{
  "scripts": {
    "dev": "next dev",
    "build": "npx prisma generate && next build",
    "start": "next start",
    "postinstall": "npx prisma generate && npx prisma migrate deploy"
  }
}
```

### 2. 推送到 GitHub

```bash
git init
git add .
git commit -m "init: 视频会议助手项目初始化"
# 在 GitHub 创建仓库后
git remote add origin <your-repo-url>
git push -u origin main
```

### 3. 在 Railway 部署

1. 访问 [railway.app](https://railway.app)，用 GitHub 登录
2. 点击 "New Project" → "Deploy from GitHub repo"
3. 选择你的仓库
4. Railway 自动检测 Next.js 项目并部署
5. 等待构建完成，获得公网 URL（如 `xxx.up.railway.app`）

### 4. 环境变量

在 Railway 项目的 Variables 中设置：

| 变量名 | 值 | 说明 |
|--------|-----|------|
| `DATABASE_URL` | `file:./prisma/dev.db` | SQLite 路径 |

### 5. 注意事项

- Railway 的磁盘是持久化的，SQLite 文件会被保留
- 如需备份，定期下载 `prisma/dev.db` 文件
- 上传的文件存储在 `public/uploads/` 目录，同样持久化

## 本地开发

```bash
# 安装依赖
npm install

# 初始化数据库
npx prisma migrate dev --name init

# 启动开发服务器
npm run dev
# → http://localhost:3000

# 查看数据库
npx prisma studio
# → http://localhost:5555
```

## 常用维护命令

```bash
# 重新生成 Prisma 客户端（修改 schema 后）
npx prisma generate

# 创建新的数据库迁移
npx prisma migrate dev --name description

# 重置数据库（清空所有数据）
npx prisma migrate reset

# 查看数据库内容
npx prisma studio
```
