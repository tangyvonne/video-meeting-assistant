# 02 — 技术选型与架构

## 技术栈

| 层级 | 技术 | 版本 | 选型理由 |
|------|------|------|----------|
| 框架 | Next.js | 14.x (App Router) | 全栈框架，前端+API一体化 |
| 语言 | TypeScript | 5.x | 类型安全，减少运行时错误 |
| 样式 | Tailwind CSS | 3.x | 原子化CSS，快速构建UI |
| 组件库 | shadcn/ui | latest | 高质量、可定制、代码可控 |
| ORM | Prisma | 5.x | 类型安全，SQLite兼容 |
| 数据库 | SQLite | 3.x | 零配置、零运维、单用户场景首选 |
| 部署 | Vercel / Railway | - | 免费部署，支持Node.js |

## 架构图

```
┌─────────────────────────────────────────────┐
│                   浏览器                      │
│         Next.js App (React SPA)              │
├─────────────────────────────────────────────┤
│  前端层 (Client Components)                   │
│  ├── 布局 (Layout: 导航 + 内容区)              │
│  ├── 页面 (Pages: / /meeting/[id] /todos)     │
│  └── 组件 (Components: 表单/卡片/列表/弹窗)    │
├─────────────────────────────────────────────┤
│  API 层 (Server Components / Route Handlers)  │
│  ├── /api/meetings    会议 CRUD               │
│  ├── /api/documents   文件上传/下载            │
│  ├── /api/transcriptions 转写记录              │
│  ├── /api/todos       待办 CRUD               │
│  └── /api/minutes     纪要生成/编辑            │
├─────────────────────────────────────────────┤
│  数据层                                        │
│  ├── Prisma ORM (类型安全的数据库操作)          │
│  ├── SQLite (单文件数据库)                      │
│  └── 文件系统 (public/uploads/ 上传文件)        │
└─────────────────────────────────────────────┘
```

## 数据流

```
用户操作 → React组件 → fetch() → Next.js API Route → Prisma → SQLite
                                                          ↓
                                                     返回 JSON → 组件更新
```

所有数据交互通过 Next.js API Routes 进行，前端组件不直接访问数据库。
