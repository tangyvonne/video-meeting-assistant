# 06 — 开发阶段划分

## 阶段总览

| 阶段 | 内容 | 预计时间 | 依赖 |
|------|------|----------|------|
| 0 | 项目初始化 + 骨架搭建 | 1天 | 无 |
| 1 | 会议列表页 | 1-2天 | 阶段0 |
| 2 | 会前准备模块 | 1天 | 阶段1 |
| 3 | 会中助手模块 | 1天 | 阶段2 |
| 4 | 会后跟进模块 | 1天 | 阶段3 |
| 5 | 待办总览页 | 1天 | 阶段4 |
| 6 | 收尾与部署 | 0.5天 | 阶段5 |

---

## 阶段 0：项目初始化

**目标**：项目能跑起来，开发环境就绪

### 任务清单
1. 使用 `create-next-app` 初始化项目，选 TypeScript + Tailwind
2. 安装 shadcn/ui CLI，初始化配置（淡蓝色主题）
3. 安装 Prisma + SQLite 依赖
4. 编写 `prisma/schema.prisma`（5 个模型）
5. 执行 `npx prisma migrate dev` 创建数据库
6. 创建 `src/lib/db.ts`（Prisma 客户端单例）
7. 搭建布局框架：`src/app/layout.tsx`（左侧导航栏 + 右侧内容区）
8. 配置 Tailwind 蓝色主题色系

### 验收标准
- `npm run dev` 启动成功
- 浏览器访问 `http://localhost:3000` 看到导航+空白内容区布局
- `npx prisma studio` 可查看空数据库

---

## 阶段 1：会议列表页

**目标**：首页展示会议列表，支持增删改

### 任务清单
1. 编写会议 CRUD 的 API 路由（`/api/meetings`）
2. 实现会议列表分段组件（今日 / 未来 / 历史 / 逾期）
3. 会议卡片组件（信息展示 + 状态标签）
4. 新建/编辑会议表单弹窗（Dialog）
5. 会议链接导入解析（POST `/api/meetings/parse-link`）
6. 点击卡片跳转到详情页（详情页暂时显示占位内容）

### 验收标准
- 可创建会议，保存到数据库
- 列表按分段正确展示
- 今日会议突出显示
- 卡片点击可跳转到 `/meeting/[id]`

---

## 阶段 2：会前准备模块

**目标**：会议详情页 Tab1 — 会前准备

### 任务清单
1. 会议详情页框架（三 Tab 切换）
2. 设备调试清单组件（默认模板，勾选交互）
3. 会议议程展示/编辑
4. 文件上传 API（`/api/documents`）
5. 文件列表 + 下载功能

### 验收标准
- 会议详情页三个 Tab 可切换
- 设备清单可逐项勾选
- 议程可编辑保存
- 文件可上传和下载

---

## 阶段 3：会中助手模块

**目标**：会议详情页 Tab2 — 会中助手

### 任务清单
1. 模拟转写区域（输入框 + 转写内容区）
2. 转写内容追加展示（带时间戳）
3. "标记重点"按钮（高亮选中的转写行）
4. "添加待办"按钮 + 表单弹窗
5. 待办 API（`/api/todos`）

### 验收标准
- 输入文字可追加到转写区
- 可标记转写行为重点（黄色高亮）
- 可添加待办并保存

---

## 阶段 4：会后跟进模块

**目标**：会议详情页 Tab3 — 会后跟进

### 任务清单
1. "结束会议"按钮（状态切换为 completed）
2. 纪要自动生成逻辑（聚合转写重点 + 待办 + 基本信息）
3. 纪要详情页（`/meeting/[id]/minutes`）
4. 纪要编辑功能
5. 纪要 API（`/api/minutes`）

### 验收标准
- 点击"结束会议"后状态变更
- 自动生成纪要草稿
- 可在纪要页编辑和保存

---

## 阶段 5：待办总览页

**目标**：跨会议待办统一管理

### 任务清单
1. 待办总览列表页（`/todos`）
2. 筛选器组件（状态筛选按钮组）
3. 待办列表（显示内容、负责人、截止日、所属会议）
4. 状态修改下拉 + 删除按钮
5. 导出 Excel 功能（`/api/todos/export`）

### 验收标准
- 待办页展示所有会议待办
- 筛选器可正常过滤
- 状态可修改，待办可删除
- Excel 文件可正常下载打开

---

## 阶段 6：收尾与部署

**目标**：上线可用

### 任务清单
1. 全局UI细节打磨（空状态占位图、loading、错误提示）
2. 页面标题、favicon
3. `npm run build` 验证生产构建
4. 部署到 Vercel/Railway
5. 公网链接验证所有功能

### 验收标准
- 生产构建无报错
- 公网可访问
- 核心流程跑通（创建会议 → 会前准备 → 会中转写+待办 → 结束会议 → 纪要生成 → 待办导出）
